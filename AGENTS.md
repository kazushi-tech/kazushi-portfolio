# AGENTS.md (Rules for Codex)

## Goal
- 最小の差分で、見た目/動作の問題を直す（“刷新”しない）

## Hard constraints (絶対)
- テキスト/ボタン文言（コピー）は変更しない
- レイアウトを全面的に作り直さない
- 変更は1〜2ファイルずつ。大規模リファクタ禁止
- 依存追加・ファイル削除・設定大変更は必ず事前に確認する
- 変更後は必ず「確認手順（どこを見ればOKか）」を書く

## Working style
- まず原因候補を2〜3個に絞って説明 → その後に修正案
- 修正は差分が小さい順に提案して、勝手に全部やらない
- 最後に変更点を箇条書きで要約する
